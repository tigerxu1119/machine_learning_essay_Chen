# 期中报告论文写作格式统一说明

本文档用于统一小组成员撰写期中报告时的格式、术语和写作口径。所有成员在提交自己负责的部分前，都应按本文档检查一遍。

## 1. 文档基本格式

- 正文字体：宋体。
- 正文字号：小四或 10.5 pt。
- 标题颜色：黑色。
- 一级标题格式：`一、方法/原理/算法介绍`、`二、相关文献综述`、`三、数据说明及描述性统计`。
- 二级标题格式：`1.1 基金异常收益的度量`、`1.2 机器学习预测框架`。
- 正文段落：使用论文写作语气，避免口语化表达。
- 段落结构：每段只表达一个核心意思，先给结论，再解释原因或方法。

## 2. 三个部分的统一结构

### 一、方法/原理/算法介绍

本部分回答“原文如何识别好基金，以及本文如何用 S12 解释这些好基金”。

建议结构：

1. 基金异常收益的度量。
2. 机器学习预测框架。
3. Top fund 的严格定义。
4. 基于 S12 持仓数据的可解释性扩展。

必须统一的写法：

- Top fund 严格沿用原文定义，即每月按照机器学习样本外预测值排序形成十分位组合，最高预测组 D10 为 top fund。
- S12 不用于重新定义好基金，只用于解释 D10 基金的真实持仓风格。
- 不写“前 10% 或前 20% 基金”，统一写“预测排序第十十分位组合 D10”。

### 二、相关文献综述

本部分不要写成逐篇论文摘要，应围绕研究逻辑组织。

建议结构：

1. 机器学习与基金绩效预测。
2. 共同基金持仓与基金经理能力。
3. 模型可解释性与投资风格。
4. 本文与已有文献的关系。

统一落脚点：

本文的贡献不是重新提出一个选基模型，而是在原文机器学习选基结果基础上，引入 S12 持仓数据解释 top fund 的投资风格和调仓逻辑。

### 三、数据说明及描述性统计

本部分回答“用了什么数据、字段是什么、如何合并、准备做哪些统计”。

建议结构：

1. 原文样本与数据来源。
2. 作者复现数据包字段。
3. S12 共同基金持仓数据字段。
4. 数据合并与样本构建。
5. 描述性统计设计。

必须统一的描述统计口径：

- 原文正文 Table 1 是 59 个特征的变量清单，不是传统均值/标准差表。
- 原文 Table 2 是基金动量变量定义。
- 原文正文报告的关键样本描述包括：1980 年 1 月至 2019 年 1 月、3,275 只基金、407,158 个 fund-month 观测、异常收益均值和中位数为 -0.03%/月、标准差为 2.00%。
- 我们自己的描述性统计可以补充 N、mean、std、p25、median、p75、min、max。

## 3. 术语统一

| 不要写 | 统一写法 |
|---|---|
| 好基金 | 机器学习识别出的 top fund / D10 基金 |
| 前 10% 或 20% 基金 | 预测排序第十十分位组合 D10 |
| 用 S12 找好基金 | 用 S12 解释 top fund 的真实持仓风格 |
| 基金收益 | 若指预测对象，应写基金异常收益 |
| alpha | 第一次出现时写“基金异常收益（alpha）” |
| 13F 数据 | 本项目主数据为 Mutual Fund Holdings - S12 |

## 4. 公式格式

公式必须使用展示公式，不要把公式写成正文中的线性字符串。

推荐格式：

```latex
\[
R^{abn}_{i,t}
=
R_{i,t}
-
F_t \hat{\beta}_{i,t-1}
\tag{2}
\]
```

写入 Word 时应转换为专业公式格式，要求：

- 公式居中。
- 公式编号右侧或公式后标注，如 `(1)`、`(2)`。
- 变量使用斜体。
- 上下标必须使用专业格式，例如 \(R^{abn}_{i,t}\)，不要写成 `R_abn_i_t`。
- 同一变量全文保持一致，不要一会儿写 `R_abn`，一会儿写 `alpha`。

本文核心公式应至少包括：

```latex
\[
R_{i,t-36:t-1}
=
\alpha_i
+
F_{t-36:t-1}\hat{\beta}_{i,t-1}
+
\eta_{i,t-36:t-1}
\tag{1}
\]
```

```latex
\[
R^{abn}_{i,t}
=
R_{i,t}
-
F_t\hat{\beta}_{i,t-1}
\tag{2}
\]
```

```latex
\[
R^{abn}_{i,t+1}
=
g(z_{i,t}, z_t; \theta)
+
\varepsilon_{i,t+1}
\tag{3}
\]
```

```latex
\[
TopFund_{i,t}
=
1\{i \in D10_t\}
\tag{4}
\]
```

```latex
\[
Style_{i,q}
=
\alpha
+
\beta TopFund_{i,q}
+
\gamma Controls_{i,q}
+
\delta_q
+
\varepsilon_{i,q}
\tag{5}
\]
```

## 5. 表格格式

表格标题统一写在表格上方，例如：

`表 1 原文复现样本描述性统计`

表格字段统一：

- 样本量写作 `N`。
- 均值写作 `mean`。
- 标准差写作 `std`。
- 中位数写作 `median`。
- 分位数写作 `p25`、`p75`。
- 最小值和最大值写作 `min`、`max`。

表格内变量名保留英文原名，中文解释可放在表注或正文中。

## 6. 数据字段写法

作者复现数据包字段按文本清单写，不必逐项解释。

核心字段：

- `abnormal_return.csv`：`md`, `wficn`, `Date`, `alpha`, `tna`, `passiveret`, `active_alpha`, `passive_alpha`, `exp_ratio`, `rret`, `stock_alpha`, `net_alpha`。
- `macro_combined.csv`：`md`, `CFNAI`, `Sentiment`。
- `alpha_imputed_chara_sentiment.npz`：`data`, `permno`, `names`, `date`。
- S12 核心持仓字段：`fundno`, `fdate`, `cusip`, `shares`。
- S12 证券辅助字段：`ticker`, `stkname`, `indcode`, `exchcd`, `prc`, `shrout1`, `shrout2`。
- 连接字段：`wficn`, `fundno`, `sdate1`, `sdate2`。

## 7. S12 扩展的统一表述

推荐表述：

本文在严格沿用原文机器学习 top fund 定义的基础上，引入 WRDS Mutual Fund Holdings - S12 数据，进一步分析 D10 基金的真实持仓风格。S12 数据不用于重新识别好基金，而是用于解释机器学习识别出的高绩效基金是否在持仓集中度、行业配置、股票风格暴露和调仓行为上具有系统性特征。

避免表述：

- “本文用 S12 数据找出好基金。”
- “持仓集中度高的基金就是好基金。”
- “我们重新定义 top fund。”
- “取前 10% 或前 20% 作为 top fund。”

## 8. 提交前检查清单

- 是否使用宋体和黑色标题。
- 是否使用“D10”定义 top fund。
- 是否明确说明 S12 只用于解释持仓风格。
- 是否避免口语化表达。
- 公式是否为专业展示公式。
- 表格变量名是否统一。
- 是否区分原文描述性统计和我们补充的描述性统计。
- 是否避免把字段解释写得过细，影响正文流畅度。
